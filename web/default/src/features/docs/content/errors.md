# 错误码

接口出错时会返回相应的 HTTP 状态码，响应体一般包含 `error` 字段说明原因。常见情况如下：

| 状态码 | 含义 | 常见原因与处理 |
|---|---|---|
| `400` | 请求有误 | 参数缺失或格式错误，检查 `model`、`messages` 等字段 |
| `401` | 鉴权失败 | 缺少或错误的密钥；检查 `Authorization: Bearer` 头 |
| `403` | 无权限 | 密钥被禁用、无该模型权限或分组受限 |
| `404` | 不存在 | 路径写错，或 `model` 指定了不存在的模型 |
| `429` | 触发限流 / 余额不足 | 降低并发后重试，或充值 / 提升额度 |
| `500` | 服务端错误 | 偶发请重试；持续出现请反馈管理员 |
| `503` | 上游不可用 | 上游暂时故障，稍后重试，网关会尝试自动切换渠道 |

## 错误响应示例

```json
{
  "error": {
    "message": "Incorrect API key provided.",
    "type": "invalid_request_error",
    "code": "invalid_api_key"
  }
}
```

## 排查建议

1. 先看 HTTP 状态码定位大类；
2. 再读 `error.message` 看具体原因；
3. 鉴权类问题核对密钥；额度 / 限流类问题查看控制台「钱包」与「使用日志」；
4. 仍无法解决时，带上请求时间与 `id` 反馈管理员，便于排查。
